How the Browser Handles a Script
пЃЅ By default (neither async nor defer is used):
1. Fetch and parse the HTML page
2. The parser encounters a <script> tag
3. If the script is external, the browser requests the script file, the HTML 
parser is blocked
4. After some time, the script is loaded and subsequently executed
5. The parser continues parsing the rest of the HTML document
пЃЅ Why? Because old developpers used to use document.write()
to generate dynamic HTML в†’ the parser needs to wait

Asynchronous Loading
пЃЅ Use async attribute to make a script run asynchronously as soon as it is 
available
пЃЅ With: script is executed asynchronously with the rest of the page, while the page 
continues the parsing
пЃЅ Without: script is fetched and executed immediately, before the browser 
continues parsing the page
пЃЅ Attn: Only for external scripts
пЃЅ Example:
пЃЅ <script src="script1.js"></script>
<script src="script2.js" async></script>
<script src="script3.js" async></script>
пЃЅ script1.js is downloaded and executed immediately, making page parser 
blocked until the execution finishes
пЃЅ script2.js and script3.js are downloaded immediately, then executed 
when available and asynchronously (i.e., in any order)
